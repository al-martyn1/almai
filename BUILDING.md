# Требуемый инструментарий

Для сборки проекта требуется:
- CMake версии 4.2 или выше;


## Зависимости для сборки с GCC (Windows)

Для сборки с использованием компилятора GCC требуется:

- компилятор GCC
- IDE VSCode
- система сборки make (MinGW) или Ninja build.

Компилятор GCC (MinGW) можно скачать по адресам:

- https://winlibs.com/
- https://www.mingw-w64.org/downloads/
- https://github.com/skeeto/w64devkit

Для поддержки локального деплоя рекомендуются следующие версии компилятора GCC:

- 7.3
- 8.1
- 13.2
- 15.2

Готовые бинарные сборки [Ninja build](https://ninja-build.org/) можно найти
на странице релизов: [https://github.com/ninja-build/ninja/releases](https://github.com/ninja-build/ninja/releases).


## Зависимости для сборки с MSVC

Требуется MS Visual Studio 2019, 2022 или 2026 с установленными инструментами и библиотеками C++.

Сборка с MS Visual Studio 2017 возможна, но не гарантируется.

# Запуск IDE

## Запуск MSVC и его настройка

В файле `.set_msvc.bat` нужно выбрать трубуемую версию MSVC, после чего запустить `_start_msvc.bat`.

Можно настроить запуск нескольких различных версий MSVC. Для этого предназначен шаблон `_start_msvcXXXX.bat.template`,
оторый следует скопировать, подставив вместо `XXXX` версию MSVC (и удалив `.template`), и настроить версию MSVC
внутри нового файла.


## Запуск VSCode

Для открытия проекта в VSCode следует запустить файл `_start_code.bat`.

# Локальный деплой

Для локального деплоя используется скрипт `deploy_local.bat`. Он ожидает наличие переменной окружения `DEPLOY_ROOT`.
Если такой переменной не существует, или она имеет пустое значение, проверяется переменная окружения `UMBA_TOOLS`.

Для деплоя по своему пути следует создать свой скрипт для деплоя.

**my_deploy.bat**

```
@set DEPLOY_ROOT=ПУТЬ\\К\\КАТАЛОГУ\\С\\ВАШЕЙ\\КОЛЛЕКЦИЕЙ\\УТИЛИТ
@call "%~dp0\deploy_local.bat"
```
