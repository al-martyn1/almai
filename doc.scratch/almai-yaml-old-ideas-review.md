# Заметки по almai.yaml (старые представления)

Сначала производится сборка зависимостей по типам, используя в начале препромпты из проектного файла 'almai.yaml', а затем из командной строки.
После того, как обработаны все заданные препропмпты и все зависимости удовлетворены, производится генерация результата.

Создаваемая утилита (назовём её 'almai') может быть запущена из произвольного каталога проекта.
Она сканирует каталоги, начиная с текущего каталога, и ищет файл с именем 'almai.yaml'.
Каталог в котором находится данный файл, является корневым каталогом проекта.

## Содержимое файла almai.yaml

Файл проекта 'almai.yaml' должен содержать:
- preprompt-types - добавление (или замена) новых типов препромптов в проект, аналогично опции `--preprompt-types`, обрабатывается до опций командной строки,
  использование символа '-' (минус) - очищает список (работает так же, как и в командной строке) - таким образом, при 
  необходимости, мы можем полностью заменить преднастроенные в конфигурационных файлах типы препромптов;
- project-preprompts - препромпты, используемые на проекте по умолчанию, без явного их задания в командной строке, обрабатывается до опций командной строки;
- project-descriptions - соответствует опции командной строки `--project-descriptions`, работает аналогично preprompt-types.
- lang - язык промптов проекта.

Файл проекта 'almai.yaml' ищется и обрабатывается после обработки "встроенных" конфигурационных файлов, но до обработки опций и конфигурационных
файлов, заданных в командной строке.


## Язык препромптов

На самом деле, это не нужно. Препромпты всегда на английском. Это тупо сэкономит токены. Но приведу старые заметки всё равно.

Язык может быть задан:
- в файле проекта 'almai.yaml' (поле lang);
- через опцию командной строки `--lang=ru` (приоритет выше).

Если язык не задан, то используется язык, задаваемый опцией `--default-lang=...`.

Приоритет способов задания языка:
- опция командной строки --lang=ru (наивысший приоритет), также должна работать короткая версия `-l`.
- поле lang в файле проекта almai.yaml.
- опция --default-lang=xx (если указана).
- встроенное значение по умолчанию – en.


## Прочие специальные файлы описаний

Надо переработать

Файл 'PROJECT.md' является специальным файлов описаний. Может быть некоторое количество
подобных файлов, кроме 'PROJECT.md', например: 'REQUIREMENTS.md', 'ROADMAP.md', 'STATE.md'.

Опция `--project-descriptions` задаёт имена файлов, которые надо включить в результирующий документ,
а также порядок их включения. Расширение '.md' не указывается. Например:
`--project-descriptions=PROJECT,REQUIREMENTS,ROADMAP,STATE`.

Файлы описаний могут содержать YAML frontmatter, но его содержимое игнорируется.

Файлы описаний вставляется по правилам расширенных препромптов, но без вводного текста.

Если файл пуст (содержит только пустые строки - пробелы и переводы строки) - он не вставляется.
Файл описаний вставляется всегда, если такой файл существует и содержит не только пустые строки.

Опция `--project-descriptions` работает аналогично опции `--preprompt-types` - так, символ минус очищает содержимое списка,
а параметр 'project-descriptions' в проектном файле работает аналогично параметру 'preprompt-types'.

Файлы описаний проекта ищуться в том же каталоге, в котором расположен файл проекта 'almai.yaml'.

Если в том же каталоге, в котором расположен файл проекта 'almai.yaml' существует каталог 
'.project-descriptions' или '.project-descriptions.almai', то используются файлы описаний, которые расположены в этих каталогах.

Если каталоги '.project-descriptions' и '.project-descriptions.almai' существуют одновременно, то используется каталог
'.project-descriptions.almai', а каталог '.project-descriptions' игнорируется.

Файлы, имена которых перечисляются в опции `--project-descriptions` (без расширения .md) должны быть полноценными markdown документами
со своими заголовками и своей структурой.


## Список "умений"

При выполнении команды --list (с необязательным указанием типов) отображаются препромпты, доступные в данный момент с учётом текущего языка 
(`--lang`, 'lang' из 'almai.yaml' или `--default-lang`). Для каждого идентификатора показывается та версия, которая была бы загружена при генерации промпта:
- если в путях поиска существует локализованная версия (например, roles/ru/cpp-dev.md), берётся информация (title, description) именно из неё;
- если локализованной версии нет, используется базовая (roles/cpp-dev.md);
- если файл отсутствует вовсе, идентификатор не отображается.

Таким образом, список всегда отражает реально доступные для использования препромпты с учётом языка.
