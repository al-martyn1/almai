# `almai-md-join` – объединение файлов в Markdown с листингами

## Обзор

`almai-md-join` – это утилита командной строки для сборки нескольких файлов (исходного кода, конфигураций, текстов) в единый Markdown-документ. Каждый файл помещается в отдельный блок кода (листинг) с возможностью указания имени файла, языка подсветки и других атрибутов. Утилита является обратной по отношению к `almai-md-split`, которая выделяет листинги из Markdown в отдельные файлы.

Основные возможности:
- Задание исходных файлов через явные пути, glob-шаблоны (с поддержкой `**` для рекурсивного обхода) или специальный синтаксис множественных масок для одного каталога.
- Чтение списка файлов из внешнего файла или стандартного ввода.
- Гибкое управление отображением имён файлов: обычный текст, заголовки, атрибуты в открывающем блоке кода.
- Автоматическое определение языка подсветки по расширению (с возможностью переопределения).
- Автоматический выбор безопасного обрамления (fence), чтобы содержимое файла не конфликтовало с маркерами блоков кода.
- Сортировка файлов по разным критериям.
- Управление путями: удаление общего префикса, отображение относительно заданной директории, только базовое имя.

## Установка

(Предполагается, что утилита поставляется в составе пакета `almai-tools`. Здесь можно описать конкретный способ установки, например, через pip или менеджер пакетов.)

Пример установки через pip:
```bash
pip install almai-tools
```

После установки команда `almai-md-join` станет доступна в командной строке.

## Использование

```bash
almai-md-join [ОПЦИИ] ПАТТЕРН [ПАТТЕРН ...]
```

### Аргументы (ПАТТЕРН)

Каждый аргумент задаёт один или несколько исходных файлов. Поддерживаются следующие формы:

1. **Явный путь к файлу**  
   Пример: `src/main.py`  
   Если путь не содержит метасимволов, он считается конкретным файлом. Относительные пути разрешаются относительно текущей рабочей директории.

2. **Glob-шаблон**  
   Пример: `src/*.py`  
   Поддерживаются стандартные метасимволы:
   - `*` – любая последовательность символов, кроме разделителя пути.
   - `?` – один любой символ.
   - `[abc]` – любой из символов `a`, `b`, `c`.
   - `**` – любое количество символов, включая разделители пути (рекурсивный обход подкаталогов).  
   Пример: `src/**/*.py` найдёт все `.py` файлы во всех подкаталогах `src`.

3. **Множественные маски для одного пути**  
   Синтаксис: `БАЗОВЫЙ_ПУТЬ/МАСКА1,МАСКА2,...`  
   Позволяет применить несколько масок к одному базовому пути без его повторения.  
   - `БАЗОВЫЙ_ПУТЬ` заканчивается разделителем (`/` или `\` в зависимости от платформы). Если разделитель отсутствует, аргумент обрабатывается как обычный glob.
   - После разделителя через запятую перечисляются маски (каждая может содержать `*`, `?`, `**` и т.д.).  
   Пример: `src/**.py,*.js,lib/**` означает:
     - все `.py` рекурсивно в `src/` (`**.py`);
     - все `.js` непосредственно в `src/` (`*.js`);
     - все файлы рекурсивно в `src/lib/` (`lib/**`).

4. **Файл со списком**  
   Аргумент, начинающийся с `@`, интерпретируется как путь к файлу, содержащему список путей (по одному на строку). Пустые строки и строки, начинающиеся с `#`, игнорируются. Каждая непустая строка может содержать путь, glob-шаблон или синтаксис множественных масок.  
   Пример: `@filelist.txt`

5. **Стандартный ввод**  
   Аргумент `-` (один дефис) означает чтение списка путей из stdin. Формат аналогичен файлу со списком.

Все способы можно комбинировать. Порядок обработки файлов определяется после раскрытия всех паттернов и применения сортировки.

### Опции

#### Ввод/вывод
| Короткая | Длинная      | Параметр | Описание |
|----------|--------------|----------|----------|
| `-o`     | `--output`   | `FILE`   | Записать результат в файл `FILE`. Если не указано, вывод идёт в stdout. |

#### Управление отображением имён файлов
| Короткая | Длинная           | Параметр                | Описание |
|----------|-------------------|-------------------------|----------|
|          | `--filename`      | `{none\|text\|title\|attr}` | Способ отображения имени файла перед листингом. По умолчанию `none`. |
|          | `--filename-format` | `FORMAT`                | Шаблон для режима `text`. Должен содержать `{filename}`. По умолчанию: `"File: {filename}"`. |
|          | `--title-level`   | `N`                     | Уровень заголовка для режима `title` (1–6). По умолчанию: 2. |

**Пояснения:**
- `none` – имя файла не выводится.
- `text` – выводится строка по шаблону `--filename-format`, затем пустая строка, затем блок кода.
- `title` – выводится заголовок уровня `N` с именем файла, затем пустая строка, затем блок кода.
- `attr` – имя файла добавляется как атрибут в открывающий блок кода, например:
  ````markdown
  ```python {filename="main.py"}
  ````

#### Управление языком подсветки
| Короткая | Длинная    | Параметр     | Описание |
|----------|------------|--------------|----------|
|          | `--lang`   | `LANG`       | Принудительно установить язык для всех блоков. Если не указано, язык определяется по расширению файла на основе встроенной таблицы. |
|          | `--lang-map` | `EXT:LANG` | Переопределить язык для конкретного расширения. Можно указывать несколько раз. Расширение указывается без точки. Пример: `--lang-map py:python --lang-map js:javascript`. |

#### Сортировка файлов
| Короткая | Длинная | Параметр                    | Описание |
|----------|---------|-----------------------------|----------|
|          | `--sort`| `КРИТЕРИЙ[:asc\|desc]`      | Задаёт порядок обработки файлов. Критерии: `none` (порядок аргументов, по умолчанию), `name` (имя файла), `ext` (расширение, затем имя), `path` (полный путь), `size` (размер), `mtime` (время модификации). Направление: `asc` (по умолчанию) или `desc`. |

#### Управление отображением путей
| Короткая | Длинная         | Параметр | Описание |
|----------|-----------------|----------|----------|
|          | `--strip-prefix`| `PATH`   | Явно указать префикс, который нужно удалить из начала каждого пути при отображении имени. Если не указано, утилита автоматически пытается найти общий префикс для всех файлов. Если общий префикс не найден (например, файлы на разных дисках), выдаётся ошибка. |
|          | `--full-path`   |          | Всегда показывать полные абсолютные пути (отменяет `--strip-prefix` и `--relative-to`). |
|          | `--relative-to` | `DIR`    | Показывать пути относительно указанной директории. По умолчанию – текущая рабочая директория. |
|          | `--basename`    |          | Показывать только имя файла (без пути). |

#### Обрамление блоков кода (fence)
| Короткая | Длинная        | Параметр                        | Описание |
|----------|----------------|---------------------------------|----------|
|          | `--fence-style`| `{backticks\|tildes\|auto}`     | Стиль обрамления: `backticks` (тройные обратные кавычки), `tildes` (тройные тильды), `auto` – автоматический выбор безопасного разделителя на основе анализа содержимого файлов. По умолчанию: `auto`. |

В режиме `auto` утилита проверяет содержимое каждого файла и выбирает такую последовательность (увеличивая длину при необходимости), которая не встретится как отдельная строка в файле, чтобы блок кода не закрылся преждевременно.

#### Прочие опции
| Короткая | Длинная          | Параметр | Описание |
|----------|------------------|----------|----------|
|          | `--ignore-missing`|          | Не выдавать ошибку, если файл не найден (просто пропускать). По умолчанию выдаётся предупреждение в stderr. |
|          | `--fail-fast`    |          | Останавливаться при первой же ошибке (например, если файл недоступен для чтения). По умолчанию утилита продолжает работу после предупреждения. |
| `-h`     | `--help`         |          | Показать справку и выйти. |
| `-V`     | `--version`      |          | Показать версию утилиты и выйти. |

### Поведение при ошибках
- Если после обработки всех паттернов не найдено ни одного файла (и не указан `--ignore-missing`), утилита завершается с кодом возврата 1 и выдаёт сообщение об ошибке.
- При невозможности прочитать файл (недостаточно прав, битая кодировка и т.п.) утилита по умолчанию выдаёт предупреждение в stderr, пропускает файл и продолжает работу. Опция `--fail-fast` меняет поведение на немедленный выход с ошибкой.
- Если автоматическое определение общего префикса не удалось (например, файлы на разных дисках в Windows) и не задан `--strip-prefix`, утилита завершается с ошибкой, требуя явного указания префикса.

## Примеры

### 1. Простейшее использование
```bash
almai-md-join script.py config.json > out.md
```
Файлы `script.py` и `config.json` будут помещены в `out.md` как блоки кода с автоопределением языка (по расширению). Имена файлов не выводятся.

### 2. Рекурсивный сбор всех `.py` файлов с именами-заголовками
```bash
almai-md-join --filename=title --title-level=3 "src/**/*.py" -o python.md
```
Все `.py` файлы из каталога `src` и его подкаталогов будут включены, перед каждым блоком будет заголовок уровня 3 с именем файла.

### 3. Использование множественных масок для одного пути
```bash
almai-md-join src/**.py,*.js,lib/**
```
Эта команда найдёт:
- все `.py` рекурсивно в `src/`;
- все `.js` непосредственно в `src/` (не рекурсивно);
- все файлы рекурсивно в `src/lib/`.

### 4. Явное указание префикса для удаления
```bash
almai-md-join --strip-prefix /home/user/project --filename=text lib/*.py
```
Имена файлов будут отображаться как `lib/script.py` вместо полного `/home/user/project/lib/script.py`.

### 5. Сортировка по расширению, затем по имени, по убыванию
```bash
almai-md-join --sort=ext:desc src/*.py src/*.js
```
Сначала пойдут файлы с расширением, идущим позже по алфавиту (например, `.py` перед `.js`), внутри каждого расширения – по убыванию имени.

### 6. Использование файла со списком
```bash
almai-md-join @manifest.txt -o result.md
```
где `manifest.txt` содержит:
```
src/main.py
src/utils.py
lib/helper.js
# это комментарий
docs/readme.md
```

### 7. Комбинация опций
```bash
almai-md-join --filename=attr --lang=python --sort=name --fence-style=auto src/**.py,test_*.py --relative-to=src -o all_code.md
```
- Имена файлов выводятся как атрибуты в открывающем блоке кода.
- Язык принудительно `python` для всех блоков (даже если расширение другое).
- Сортировка по имени файла.
- Автоматический выбор безопасного fence.
- Пути отображаются относительно каталога `src`.

## Замечания

- Все входные файлы предполагаются в кодировке UTF-8. Если встречаются некорректные последовательности, они заменяются специальным символом (во избежание сбоев).
- При использовании `**` в glob рекурсивный поиск работает на всех платформах, но имейте в виду, что в больших деревьях каталогов это может занять время.
- Если в именах файлов встречаются запятые, их нельзя использовать в синтаксисе множественных масок (запятая считается разделителем). В таких случаях файл следует указывать отдельным аргументом или через @-файл.
- Утилита предназначена для пакетной обработки; для интерактивного использования можно комбинировать с `find` и передачей списка через `-`.

## Сравнение с `almai-md-split`

`almai-md-split` выполняет обратную операцию: из Markdown-документа извлекает листинги и сохраняет их в отдельные файлы. Вместе эти утилиты образуют полный цикл работы с листингами: можно разбить документ на файлы, отредактировать их, а затем собрать обратно.
