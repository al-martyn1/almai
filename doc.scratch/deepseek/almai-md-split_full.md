# Задача: реализовать утилиту almai-md-split согласно техническому заданию

Ты — опытный Python-разработчик и архитектор ПО. Тебе необходимо создать полноценный проект утилиты `almai-md-split`, следуя приложенному ниже техническому заданию (ТЗ).

## Требования к результату

Результат должен быть представлен **в виде одного markdown-документа**, который содержит все файлы проекта. Каждый файл должен быть оформлен как отдельный листинг, перед которым расположена строка с именем этого файла (включая относительный путь от корня проекта), в точности так, как это описано в ТЗ для входных markdown-документов. Например:

```markdown
"src/almai_md_split/__init__.py"
```
```python
# содержимое файла
```

Имя файла может быть обрамлено кавычками, двойными звёздочками и т.п. — допускается любой из вариантов, перечисленных в ТЗ (двойные/одинарные кавычки, **жирный**, *курсив*, `код`). Главное, чтобы оно было непосредственно перед листингом и между ними не было пустых строк.

Все файлы проекта должны быть включены в документ в логическом порядке (например, сначала основные модули, потом тесты, затем конфигурационные файлы). Не забудь добавить `pyproject.toml` для упаковки проекта, файл словаря по умолчанию (`almai-md-split.dict`), а также README с краткой инструкцией.

## Содержание проекта

Проект должен полностью соответствовать приложенному ТЗ. Особое внимание удели:
- Архитектуре: разделение на логические модули (парсер, обработчик имён, работа со словарями, нормализация, разрешение коллизий, CLI).
- Использованию встроенных библиотек по возможности, допустимо использовать `platformdirs` и `unidecode` (если нужно).
- Тестам с использованием `pytest`: модульные, интеграционные и CLI-тесты.
- Корректной обработке ошибок и предупреждений с соответствующими префиксами.
- Кроссплатформенности (Windows, Linux, Unix-подобные).
- Полной поддержке всех требований ТЗ: вложенные листинги, экранирование, нормализация имён, разрешение коллизий, словари, зарезервированные имена Windows и т.д.

## Формат вывода

Твой ответ должен содержать **только** markdown-документ с перечисленными файлами. 
Никаких дополнительных пояснений, комментариев или вступлений не требуется. 
Сам документ должен быть оформлен как обычный markdown (расширение .md) и готов к использованию.
