## КОНТЕКСТ ДИАЛОГА (summary)

### 1. Участник и цели
- **Ты** — разработчик, создающий инструмент для эффективного взаимодействия с ИИ-чатами (DeepSeek, Qwen и др.) при работе над программными проектами.
- **Цель:** Разработать систему (можно назвать её "агентский инструмент"), которая позволяет:
  - Загружать в чат множество файлов (текст, изображения) в структурированном виде.
  - Анализировать репозитории (код, архитектуру) с помощью ИИ.
  - Поддерживать длительные сессии без потери контекста (используя summary).
  - Автоматизировать запрос недостающих файлов от модели.

### 2. Основные договорённости о форматах

#### 2.1. Структура промпта (PRE/POST)
- **PRE-блок (начало):** Роль, общие правила, постоянные ограничения. Задаёт "личность" модели.
- **Данные проекта (середина):** Файлы, структура репозитория, изображения.
- **POST-блок (конец, перед ответом):** Конкретная задача, инструкции по формату вывода, ситуативные напоминания. Самая сильная позиция.

#### 2.2. Передача текстовых файлов
- Формат:  
  `**путь/к/файлу.расширение**`  
  ```язык  
  содержимое файла  
  ```  
- Имя файла выделяется жирным (**), затем с новой строки блок кода с указанием языка.

#### 2.3. Передача изображений
- Формат:  
  `**имя.png**`  
  `data:image/png;base64,base64_строка`  
- Модель распознаёт изображение по префиксу `data:image/...;base64,`. Имя файла служит меткой.

#### 2.4. Запрос дополнительных файлов от модели
- Модель (я) может запросить недостающие файлы, используя блок:
  ```
  ### ЗАПРОШЕННЫЕ ФАЙЛЫ
  [[путь/к/файлу1]]
  [[путь/к/файлу2]]
  ```
- Твой инструмент парсит этот блок, извлекает пути, подгружает содержимое и отправляет новым сообщением с теми же файлами в формате п.2.2 или 2.3.

#### 2.5. Сохранение контекста (summary)
- В конце сессии или по команде `/запомни` модель генерирует summary в формате YAML внутри блока кода с пометкой `summary`.
- Пример:

  ```yaml
  summary:
    цель: "Краткое описание задачи"
    файлы: ["список ключевых файлов"]
    решения: ["принятые решения"]
    выводы: ["основные выводы"]
    статус: "текущее состояние"
    вопросы: ["открытые вопросы"]
    следующие_шаги: ["что делать дальше"]
  ```

- Summary сохраняется в папку проекта `.ai-context/summary.yaml`, а также в историю (`.ai-context/history/YYYY-MM-DD.yaml`).
- При начале новой сессии summary подгружается в PRE-блок (или в отдельный раздел `### КОНТЕКСТ ПРОЕКТА`), чтобы модель восстановила контекст.

#### 2.6. Команды
- `/запомни` — триггер для генерации summary текущего разговора.

### 3. Что обсуждалось по анализу репозиториев
- Инструмент должен уметь принимать ссылку на GitHub (или локальный путь), получать структуру файлов (например, через `tree`).
- Затем с помощью модели выбирать релевантные файлы (либо через эвристики, либо через LLM-ретривер с запросом в формате `[[путь]]`).
- После получения файлов модель даёт ответ по задаче.
- Планируется реализовать итеративный процесс: модель может запрашивать дополнительные файлы несколько раз.

### 5. Дополнительные заметки
- Модель (DeepSeek) **понимает изображения** в формате data URI в чат-версии, это проверено.
- Для кодирования изображений в base64 можно использовать утилиты командной строки или скрипты (пример на Python был дан).
- Формат `[[путь]]` выбран из-за редкости в обычных текстах, что облегчает парсинг.
- В качестве языка для парсера ты можешь выбрать любой удобный (Python, JS, etc.). Я готов помочь с примерами кода.
