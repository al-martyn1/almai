Да, вы можете запустить проект непосредственно из исходников без установки пакета. Вот несколько способов:

### 1. Запуск как модуль Python (рекомендуемый способ)
Перейдите в корневую директорию проекта (ту, где находится папка `src`) и выполните:
```bash
python -m almai_md_split путь/к/файлу.md
```
При этом Python будет искать пакет `almai_md_split` в текущем каталоге (благодаря наличию `src`). Убедитесь, что все зависимости установлены:
```bash
pip install platformdirs unidecode
```
(можно установить только их, без установки самого пакета).

### 2. Запуск через явный путь к `__main__.py`
```bash
python src/almai_md_split/__main__.py путь/к/файлу.md
```
Это тоже сработает, но модуль будет считать себя скриптом, и импорты внутри пакета могут работать нестабильно, если не настроен `PYTHONPATH`. Лучше использовать первый способ.

### 3. Установка в режиме разработки (editable install)
Если вы планируете часто использовать утилиту и вносить изменения, удобно установить её в режиме "редактирования":
```bash
pip install -e .
```
Это создаст ссылку на исходный код в site-packages, и команда `almai-md-split` станет доступна глобально (в текущем окружении). При этом любые изменения в коде будут сразу видны без переустановки.

### 4. Использование виртуального окружения
Рекомендуется создать виртуальное окружение, чтобы изолировать зависимости:
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows
pip install platformdirs unidecode
python -m almai_md_split document.md
```

Все эти способы не требуют полноценной установки пакета в систему, только наличие Python и зависимостей. При первом запуске убедитесь, что словарь по умолчанию (`conf/almai-md-split.dict`) доступен. Если вы запускаете из корня проекта, он будет найден автоматически через механизм `platformdirs` (файл скопируется в системную или пользовательскую директорию конфигов) или через fallback-путь `../conf` относительно исполняемого скрипта. В режиме разработки лучше явно указать словарь через `--dict`, если он не находится автоматически.

Пример:
```bash
python -m almai_md_split --dict=conf/almai-md-split.dict input.md
```
