Если нужно подчеркнуть, что человек **хорошо** умеет разбираться в чужом коде (а не просто делает это по необходимости), то используют такие названия:

**Профессиональные / для резюме:**
- **Специалист по сопровождению / Maintenance Developer** — классика, но звучит немного узко.
- **Легаси-инженер / Legacy Engineer** — подчёркивает умение работать с устаревшей или сложной кодовой базой.
- **Инженер по рефакторингу / Refactoring Engineer** — фокус на улучшении чужого кода.
- **Программист-аналитик / Programmer Analyst** — делает акцент на анализе кода, а не только на написании нового.

**Сленговые / ёмкие:**
- **Код-археолог / Code Archaeologist** — если код старый, запутанный, без документации.
- **Детектив / Code Detective** — когда нужно найти сложную ошибку или понять логику без автора.
- **Сапёр** — если чужой код «минный» и требует осторожности.
- **Дебаггер-эксперт / Debugging Expert** — делает упор на поиск и устранение проблем.

**Как назвать должность, чтобы это звучало привлекательно:**
Вместо одного слова часто используют описание:
> *Senior Software Engineer with strong expertise in legacy code analysis, refactoring, and technical debt reduction.*

Или коротко: **«разработчик, специализирующийся на работе с унаследованным кодом»**.

Если же нужно просто в разговоре похвалить коллегу — часто говорят: *«он отлично читает чужой код»* или *«у него сильный скилл обратного инжиниринга кода»*.
