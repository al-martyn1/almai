# Техническое задание на утилиту `almai-md-join`

## 1. Общие сведения

### 1.1 Наименование
Утилита `almai-md-join` (входит в набор утилит `almai`).

### 1.2 Назначение
Утилита предназначена для сборки нескольких файлов (исходного кода, конфигураций, текстовых документов и т.п.) в единый Markdown-документ. Каждый исходный файл помещается в отдельный блок кода (листинг) с возможностью указания имени файла, языка подсветки и других атрибутов. Утилита является обратной по отношению к `almai-md-split`, которая выделяет листинги из Markdown в отдельные файлы.

### 1.3 Область применения
- Документирование исходного кода.
- Создание отчётов, включающих фрагменты кода.
- Подготовка материалов для публикации (например, в блогах, вики).
- Автоматизация сборки технической документации.

### 1.4 Термины и определения
- **Листинг** — блок кода в Markdown, ограниченный тройными обратными кавычками (```` ``` ````) или тильдами (`~~~`).
- **Fence** — последовательность символов, открывающая и закрывающая блок кода.
- **Glob** — шаблон для поиска файлов с использованием метасимволов (`*`, `?`, `[ ]`, `**`).
- **Маска** — часть glob-выражения после базового пути в синтаксисе множественных масок.

## 2. Функциональные требования

### 2.1 Общее описание работы
Утилита принимает на вход список файлов (заданных явно, через glob-паттерны или специальный синтаксис), обрабатывает их в заданном порядке и формирует на стандартном выходе (или в указанном файле) Markdown-документ. Для каждого файла в документ включается:
- (опционально) имя файла в заданном формате;
- открывающий fence с указанием языка (если определён) и дополнительных атрибутов;
- содержимое файла (без изменений);
- закрывающий fence.

### 2.2 Входные данные
#### 2.2.1 Способы указания файлов
Утилита должна поддерживать следующие способы задания исходных файлов (в порядке обработки):
1. **Явное перечисление** — аргументы командной строки, не содержащие метасимволов glob, трактуются как пути к конкретным файлам. Если путь относительный, он разрешается относительно текущей рабочей директории.
2. **Glob-паттерны** — аргументы, содержащие символы `*`, `?`, `[ ]`, `**`, обрабатываются с помощью механизма glob. Поддерживается рекурсивный обход при использовании `**`.
3. **Синтаксис множественных масок для одного пути** — аргумент вида `БАЗОВЫЙ_ПУТЬ/МАСКА1,МАСКА2,...` позволяет применить несколько масок к одному базовому пути без его повторения.
   - Базовый путь отделяется последним разделителем (в зависимости от платформы `/` или `\`). Если разделитель отсутствует, аргумент обрабатывается как обычный glob-паттерн.
   - После разделителя через запятую перечисляются маски. Каждая маска может содержать glob-метасимволы, включая `**`.
   - Пробелы вокруг запятых не допускаются (запятая является строгим разделителем).
   - Пример: `src/**.py,*.js,lib/**` означает: в каталоге `src/` найти все `.py` рекурсивно (`**.py`), все `.js` только в самом `src` (`*.js`) и все файлы рекурсивно в `src/lib/` (`lib/**`).
4. **Файл со списком** — аргумент, начинающийся с символа `@`, интерпретируется как путь к файлу, содержащему список путей (по одному на строку). Пустые строки и строки, начинающиеся с `#` (комментарии), игнорируются. Каждая непустая строка может содержать путь, glob-паттерн или синтаксис множественных масок.
5. **Стандартный ввод** — аргумент `-` (один дефис) означает чтение списка путей из stdin. Формат аналогичен файлу со списком (по одному пути на строку).

Все указанные способы можно комбинировать в одном вызове. Порядок обработки файлов определяется после раскрытия всех паттернов и применения сортировки (см. п. 2.3.4).

#### 2.2.2 Обработка путей
- Все пути, полученные из аргументов, нормализуются (разрешаются относительно текущей директории, если они относительные, и приводятся к абсолютным для внутреннего использования).
- Если путь указывает на несуществующий файл, поведение зависит от опции `--ignore-missing` (по умолчанию выдаётся предупреждение в stderr, и файл пропускается). При использовании glob-паттернов, не найдя ни одного файла, утилита также выдаёт предупреждение (если не указано `--ignore-missing`).

### 2.3 Опции командной строки
Утилита должна поддерживать следующие опции (регистрозависимые, короткие и длинные формы):

#### 2.3.1 Ввод/вывод
| Короткая | Длинная         | Параметр | Описание |
|----------|-----------------|----------|----------|
| `-o`     | `--output`      | `FILE`   | Записать результат в файл `FILE`. Если не указано, вывод идёт в stdout. |

#### 2.3.2 Управление отображением имён файлов
| Короткая | Длинная              | Параметр                | Описание |
|----------|----------------------|-------------------------|----------|
|          | `--filename`         | `{none\|text\|title\|attr}` | Способ отображения имени файла (по умолчанию `none`). |
|          | `--filename-format`  | `FORMAT`                | Шаблон для режима `text`. Должен содержать `{filename}`. По умолчанию: `"File: {filename}"`. |
|          | `--title-level`      | `N`                     | Уровень заголовка для режима `title` (1–6). По умолчанию: 2. |

#### 2.3.3 Управление языком подсветки
| Короткая | Длинная         | Параметр     | Описание |
|----------|-----------------|--------------|----------|
|          | `--lang`        | `LANG`       | Принудительно установить язык для всех блоков. Если не указано, язык определяется по расширению файла на основе встроенной таблицы соответствий. |
|          | `--lang-map`    | `EXT:LANG`   | Переопределить язык для конкретного расширения. Можно указывать несколько раз. Расширение указывается без точки. |

#### 2.3.4 Сортировка файлов
| Короткая | Длинная   | Параметр                    | Описание |
|----------|-----------|-----------------------------|----------|
|          | `--sort`  | `КРИТЕРИЙ[:asc\|desc]`      | Задаёт порядок обработки файлов. Критерии: `none` (порядок аргументов, по умолчанию), `name` (имя файла), `ext` (расширение, затем имя), `path` (полный путь), `size` (размер), `mtime` (время модификации). Направление: `asc` (по умолчанию) или `desc`. |

#### 2.3.5 Управление отображением путей
| Короткая | Длинная          | Параметр  | Описание |
|----------|------------------|-----------|----------|
|          | `--strip-prefix` | `PATH`    | Явно указать префикс, который нужно удалить из начала каждого пути при отображении имени. Если не указано, утилита автоматически пытается найти общий префикс для всех файлов. Если общий префикс не найден (например, файлы на разных дисках), выдаётся ошибка. |
|          | `--full-path`    |           | Всегда показывать полные абсолютные пути (отменяет `--strip-prefix` и `--relative-to`). |
|          | `--relative-to`  | `DIR`     | Показывать пути относительно указанной директории. По умолчанию — текущая рабочая директория. |
|          | `--basename`     |           | Показывать только имя файла (без пути). |

#### 2.3.6 Обрамление блоков кода
| Короткая | Длинная         | Параметр                        | Описание |
|----------|-----------------|---------------------------------|----------|
|          | `--fence-style` | `{backticks\|tildes\|auto}`     | Стиль обрамления: `backticks` (тройные обратные кавычки), `tildes` (тройные тильды), `auto` — автоматический выбор безопасного разделителя на основе анализа содержимого файлов. По умолчанию: `auto`. |

#### 2.3.7 Прочие опции
| Короткая | Длинная           | Параметр | Описание |
|----------|-------------------|----------|----------|
|          | `--ignore-missing`|          | Не выдавать ошибку, если файл не найден (просто пропускать). По умолчанию выдаётся предупреждение в stderr. |
|          | `--fail-fast`     |          | Останавливаться при первой же ошибке (например, если файл недоступен для чтения). По умолчанию утилита продолжает работу после предупреждения. |
| `-h`     | `--help`          |          | Показать справку и выйти. |
| `-V`     | `--version`       |          | Показать версию утилиты и выйти. |

### 2.4 Выходные данные
Утилита генерирует Markdown-документ, состоящий из последовательности блоков (листингов) для каждого исходного файла в порядке, определённом сортировкой. Формат вывода:

- **Если `--filename=none`**: только блок кода.
- **Если `--filename=text`**: строка, сформированная по шаблону `--filename-format` (с заменой `{filename}`), затем пустая строка, затем блок кода.
- **Если `--filename=title`**: заголовок уровня `N` (из `--title-level`) с текстом — именем файла, затем пустая строка, затем блок кода.
- **Если `--filename=attr`**: открывающий fence дополняется атрибутом `filename` в фигурных скобках: например, ` ```python {filename="file.py"} `.

Блок кода:
- Открывающий fence: согласно `--fence-style` (после автоматического выбора, если `auto`), затем пробел, затем идентификатор языка (если определён). Для режима `attr` после языка через пробел добавляется `{filename="..."}`.
- Содержимое файла (передаётся без изменений).
- Закрывающий fence (такой же последовательности, как открывающий).

Если файл пуст, блок кода всё равно создаётся (пустое содержимое между fences).

### 2.5 Поведение в особых ситуациях

#### 2.5.1 Отсутствие файлов
Если после обработки всех паттернов не найдено ни одного файла (и не указан `--ignore-missing`), утилита завершается с кодом возврата 1 и выдаёт сообщение об ошибке в stderr.

#### 2.5.2 Ошибки доступа к файлам
При невозможности прочитать файл (например, недостаточно прав) утилита:
- Если не указан `--fail-fast`, выдаёт предупреждение в stderr, пропускает этот файл и продолжает работу.
- Если указан `--fail-fast`, завершается с кодом возврата 1 после первого же такого файла.

#### 2.5.3 Автоматическое определение префикса
Если не указаны `--strip-prefix`, `--full-path` и `--basename`, утилита пытается найти общий префикс для всех абсолютных путей обрабатываемых файлов. Алгоритм:
1. Все пути приводятся к абсолютным.
2. Определяется общая начальная часть (набор компонентов пути, общий для всех файлов).
3. Если общая часть длиннее корневого каталога (для Unix — не просто `/`, для Windows — не просто `C:\`), она отрезается от каждого пути при отображении имени. Иначе (если общего префикса нет или он равен корню) утилита завершается с ошибкой, требуя явного указания `--strip-prefix`.

#### 2.5.4 Конфликт fence с содержимым
В режиме `--fence-style=auto` для каждого файла выполняется анализ его содержимого с целью выбрать такую последовательность fence, которая не встретится как отдельная строка в файле (что могло бы преждевременно закрыть блок). Алгоритм:
- Начинаем с трёх обратных кавычек (```` ``` ````).
- Проверяем, содержится ли в файле строка, состоящая ровно из этой последовательности (возможно с пробелами в начале или конце — по усмотрению реализации, но безопаснее считать точное совпадение).
- Если содержится, увеличиваем длину fence на 1 (четыре кавычки) и проверяем снова. Продолжаем до тех пор, пока не найдём безопасную длину или не достигнем предела (например, 10).
- Если ни одна длина обратных кавычек не безопасна, переключаемся на тильды (`~~~`) и повторяем процесс.
- Если и тильды не помогают, выбирается максимальная проверенная длина обратных кавычек (с выдачей предупреждения).

#### 2.5.5 Кодировки
Все входные файлы предполагаются в кодировке UTF-8. Если файл содержит некорректные последовательности UTF-8, утилита должна либо прочитать его как бинарные данные (побайтово) и выдать предупреждение, либо завершиться с ошибкой (в зависимости от политики). Рекомендуется читать в текстовом режиме с обработкой ошибок заменой (`errors='replace'`), чтобы избежать сбоев.

### 2.6 Требования к платформе
Утилита должна работать в следующих окружениях:
- Linux (любой дистрибутив с поддержкой Python 3.8+ или аналогичной среды).
- macOS (10.15+).
- Windows (10/11, Server 2016+).

Особенности:
- Корректная обработка путей с разделителями `/` и `\` в зависимости от платформы.
- Поддержка символов Unicode в именах файлов и содержимом.
- При использовании `**` в glob, рекурсивный поиск должен работать на всех платформах.

### 2.7 Требования к производительности
- Утилита должна эффективно обрабатывать большое количество файлов (тысячи) и большие файлы (мегабайты).
- Не допускается загрузка всех файлов в память одновременно (потоковая обработка: чтение файла -> запись в вывод).

### 2.8 Требования к документированию
- Полное руководство пользователя (man-страница или встроенная справка `--help`) с описанием всех опций и примеров.
- Примеры использования в различных сценариях.
- Описание формата входных данных (синтаксис множественных масок, файла со списком).

## 3. Интерфейс командной строки (резюме)

```
almai-md-join [OPTIONS] PATTERN [PATTERN ...]

PATTERN:
  path/to/file
  glob/pattern/with/*.ext
  base/path/**.py,*.js,sub/**    (multiple masks for one base path)
  @filelist.txt                   (read list from file)
  -                               (read list from stdin)

OPTIONS:
  -o, --output FILE               output file (default: stdout)
  --filename {none|text|title|attr}
                                  filename display mode (default: none)
  --filename-format FORMAT        format for 'text' mode (default: "File: {filename}")
  --title-level N                 heading level for 'title' (1-6, default: 2)
  --lang LANG                     force language for all blocks
  --lang-map EXT:LANG             override language for extension
  --fence-style {backticks|tildes|auto}
                                  code block fence style (default: auto)
  --sort CRIT[:asc|desc]          sort order (none,name,ext,path,size,mtime)
  --strip-prefix PATH             explicit prefix to strip from paths
  --full-path                     show full absolute paths
  --relative-to DIR               show paths relative to DIR (default: current)
  --basename                      show only filename
  --ignore-missing                skip missing files without warning
  --fail-fast                     exit on first error
  -h, --help                      show this help
  -V, --version                   show version
```

## 4. Критерии приёмки

1. Утилита корректно собирает файлы в Markdown-документ с соблюдением указанных опций.
2. Обрабатываются все способы задания файлов (явные пути, glob, множественные маски, файл со списком, stdin).
3. Правильно определяется язык по расширению (с возможностью переопределения).
4. Автоматически выбирается безопасный fence в режиме `auto`.
5. Корректно работает сортировка по всем критериям.
6. Правильно обрезаются префиксы (автоматически или по заданию).
7. Обрабатываются ошибки (отсутствие файлов, недоступность) согласно опциям.
8. Утилита работает на всех целевых платформах.
9. Справка `--help` полная и понятная.

## 5. Возможные расширения (не обязательны для первой версии)
- Поддержка YAML front matter в выходном документе.
- Возможность вставлять произвольный текст между листингами.
- Генерация оглавления.
- Поддержка других форматов входных файлов (например, изображений — вставка как base64).
