У меня есть набор файлов. В нём описаны умения, распределённые по ролям и языкам, на которых они описаны.

Они лежат в файловой системе по каталогам `$РОЛЬ/.ru/умение.md` и `$РОЛЬ/.en/умение.md`.

Я решил не задавать жестко умения по ролям, а назначать умения ролям динамически. 
В связи с этим мне нужен один каталог с файлами умений (вернее, два - для умений, описанных на русском, и для умений, описанных на английском).

Также, мне надо отделить умения от конкретных инструкций и поместить их отдельно.

Итоговая структура каталогов такая:
- skills
  - .ru - в данном каталоге находятся умения, описанные на русском
  - .en - в данном каталоге находятся умения, описанные на русском
- instructions
  - .ru - в данном каталоге находятся инструкции, описанные на русском
  - .en - в данном каталоге находятся инструкции, описанные на русском

Файлы объединены в единый markdown документ, в котором представлены в виде блоков кода (листингов).
Для обозначения блоков кода используются fence вида \`\`\` или ~~~ (но длина fence может быть увеличена, 
если в файле, который вставлен в листинг, уже используются такие fence).

Перед каждым листингом приведено имя файла, листинг которого приводится. Оно оформлено **жирным**, и расположено 
непосредственно перед блоком кода (без пустых строк между ними).
